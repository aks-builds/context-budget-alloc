# Contributing

Thanks for considering a contribution.

1. Fork and clone the repo.
2. `npm install`
3. `npm test` before opening a PR; `npm run lint` and `npm run typecheck` too.
4. `npm run test:coverage` if you are adding non-trivial logic - keep new code covered.
5. Keep changes focused - one logical change per PR.
6. Update `CHANGELOG.md` under `[Unreleased]` for user-facing changes.
